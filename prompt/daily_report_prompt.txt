**⚠️ 首先确认当前日期（必须最先执行）**：
执行任务时，必须通过执行系统命令 `date` 获取当前的北京时间日期和星期，并以此确定报告标题中的日期。**严禁使用训练数据中的日期或猜测当前日期**。

**重要执行规则**：
- 结构化数据优先从预抓取 JSON 文件获取（见下方「预抓取数据源说明」）。两者均不可得时标注「数据暂不可得」。
- 在报告开头注明查询时间（北京时间）和当日星期。
- **今日定性导语（必须输出）**：在注明查询时间与星期后，用一句话给出「市场定性 + 主线 + 关键风险/事件」，置于宏观日历之前。如「风险偏好回升，科技领涨、高股息防御；关注今晚美国CPI」。
- 所有涨跌幅使用 ↑（涨）/ ↓（跌）箭头配合百分比表示，如 **↑1.25%**、**↓0.83%**。箭头本身已表达涨跌方向，无需额外颜色标注。
- 关键数据用 **加粗** 突出。
- 使用结构化 Markdown 格式，各级标题清晰。
- **数据来源标注**：各板块末尾汇总标注即可（见上方「数据来源」声明），无需逐条写来源或发布时间。
- 如果某项数据无法获取，明确标注「数据暂不可得」并说明原因，不要编造。
- **严格去重**：行业洞察中所有内容，包括 TOP 10 新闻、QDII 追踪、以及各板块新闻，均不得与前文已提及的任何信息重复。前文出现过的市场数据、驱动因素、板块/个股动态、基金信息等一律不再在行业洞察出现。确保全篇无重复内容。
- **信息增量检验**：每新增一段/子节前自检，内容是否为前文未覆盖的新事实或新视角？若与「个人持仓」「市场全景」已述内容重复（如招行分红、红利低波涨幅），必须合并或删除，不得另设子节。
- **防御高股息口径合并**：长江电力与红利低波同属「防御+高股息」，相关板块叙述（持仓聚焦、资金流向）统一口径，避免重复（呼应信息增量检验）。

**执行模式判断（必须先判断）**：

## ▸ 精简模式（周日、周一 或 前一日市场全面休市时执行）

当日报告仅包含以下板块，不生成市场行情、估值和持仓动态内容。

### 一、行业洞察

- 全球 TOP 10（60-80 字/条）
- A 股资金流向回顾（读取 `data_industry.json` → 资金_净流入TOP5/资金_全市场净流入_亿字段，周末/节前最后几个交易日资金动向总结）
- 科技板块动态（周末/节前科技要闻复盘，无重大新闻则注明「无明显异动」）

### 二、深度观察专栏

**周日/周一主题**（AI 根据当日数据自动选择最佳主题）：
- **A 股资金流向趋势总结**（5 天数据拉通分析，板块轮动全景）
- **全球宏观一周回顾**（利率预期变化、地缘风险演变、下周展望）
- **红利资产专题**（高股息板块资金动向、分红公告汇总、红利低波股息率跟踪）
- **QDII溢价趋势**（跨境ETF溢价率周度变化、套利机会提示）
- **银行股专题**（招商银行A/H走势、银行板块资金流、行业监管动态）

500-800 字，结构：
1. **事件概述**：事实 + 关键数据
2. **深层分析**：≥ 2 个维度（经济逻辑、地缘政治、行业格局等）
3. **影响与前瞻**：对市场、持仓的连锁影响

要求：有观点有依据，引用具体事件或数据支撑。
